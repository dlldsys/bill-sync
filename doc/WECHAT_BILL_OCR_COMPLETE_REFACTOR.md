# 微信账单OCR识别逻辑完整重构总结

## 修改概述

根据用户反馈，对微信账单OCR识别逻辑进行了全面重构，解决了"日期识别不对，数据数量也不对"的核心问题，并实现了正确的加减号金额识别和上下结构解析。

## 🎯 用户反馈问题

### 原始问题
1. **日期识别不对** - 跨行匹配导致日期时间混乱
2. **数据数量不对** - 无关文本被误识别为账单记录
3. **商家名称错误** - 识别为预设名称而非图片实际内容
4. **金额格式错误** - 应该使用加减号而非¥符号

### 用户明确要求
- **结构**: 图标+上下结构的商品名称和日期时间+金额
- **商家**: 应该是图片内容，不是预设名称
- **金额**: 必须有+或-符号，+号收入，-号支出
- **有效性**: 只有带加减号的金额才是有效数据

## 🔧 核心重构内容

### 1. 识别策略根本改变

#### 修改前 (错误方式)
```typescript
// 逐行检测商家，后续行查找时间和金额
for (let i = 0; i < lines.length; i++) {
  const merchant = detectMerchant(lines[i]);
  // 在后续行查找时间和金额...
}
```

#### 修改后 (正确方式)
```typescript
// 上下结构识别：上行商家，下行时间金额
for (let i = 0; i < lines.length; i++) {
  const merchantInfo = parseMerchantLine(lines[i]);
  if (merchantInfo) {
    const amountInfo = parseTimeAmountLine(lines[i + 1]);
    // 组合完整记录...
  }
}
```

### 2. 正确的微信账单结构理解

#### 实际结构
```
🍜 实际商家名称        ← 图标 + 商家名称 (上行)
   14:30    +45.60   ← 时间 + 金额 (下行)

☕ 实际商家名称        ← 图标 + 商家名称 (上行)  
   12:00    -38.00   ← 时间 + 金额 (下行)
```

#### 识别逻辑
- **上行**: 检测图标+实际商家名称
- **下行**: 检测时间+加减号金额
- **组合**: 两行信息组合成一条完整记录
- **跳过**: 不符合格式的行被过滤

### 3. 加减号金额识别

#### 支持的有效格式
```typescript
// 收入格式 (+号)
"14:30    +45.60"  ← 时间 + 收入金额
"+45.60"             ← 仅收入金额

// 支出格式 (-号)  
"14:30    -38.00"  ← 时间 + 支出金额
"-38.00"             ← 仅支出金额
```

#### 无效格式 (被过滤)
```typescript
"14:30    ¥45.60"  ← 不识别¥符号
"45.60"             ← 无加减号，无效
"¥45.60"            ← 不识别¥符号
```

### 4. 商家名称提取优化

#### 修改前 (限制性)
```typescript
// 只识别预设商家名称
const merchantMatch = line.match(/^([\u4e00-\u9fa5]{2,15})$/);
```

#### 修改后 (灵活性)
```typescript
// 提取图片中的实际内容
const emojiMatch = line.match(/^[\u{1F300}-\u{1F9FF}\u{2600}-\u{26FF}\u{2700}-\u{27BF}]\s*(.+)$/u);
const merchantMatch = line.match(/^([\u4e00-\u9fa5].+)$/);
```

### 5. 收入支出类型判断

#### 自动类型识别

  billType: amountInfo.type === 'income' ? 'income' : 'expense', // 根据符号设置类型
