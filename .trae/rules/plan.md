# 项目进度记录规则

## 核心要求

1. **每次关键修改必须记录**：
   - 新增重要功能
   - 修改核心逻辑
   - 修复重大bug
   - 数据库变更
   - API接口变更

2. **记录格式**：
   在 `doc/PROGRESS.md` 文件中追加记录：
   
   ```
   ## [日期] - 进度更新
   
   ### 新增/修改
   - 描述
   
   ### 状态
   - 进行中 / 已完成 / 待测试
   ```

3. **读取检查**：
   - 每次对话开始时，读取 `doc/PROGRESS.md` 了解项目当前进度
   - 每次执行关键操作前，确认进度记录已更新

## 记录时机

- 完成重要功能模块时
- 进行破坏性修改前
- 每天工作结束时
- 项目里程碑达成时

## 记录原则

- 简洁明了，描述清楚做了什么
- 标注状态：进行中 / 已完成 / 待测试 / 已部署
- 包含相关文件路径
- 遇到问题也要记录
