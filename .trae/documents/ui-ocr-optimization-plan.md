# UI 重新绘制和 OCR 识别优化计划

## 问题分析

### 当前 UI 问题
1. **UI 风格与参考图片不符**：当前使用蓝紫色渐变主题，参考图片显示的是**青色/蓝绿色**主题
2. **布局结构差异**：参考图片显示更简洁的卡片式布局，当前实现较为复杂
3. **颜色系统不匹配**：参考图片使用浅色背景 + 青色强调色，当前使用深色渐变

### 当前 OCR 问题
1. **数据条数混乱**：识别结果数量不准确
2. **识别数量错误**：多张图片批量识别时数据合并混乱
3. **日期识别混乱**：没有正确提取图片中的日期，默认使用当前日期

## 实施计划

### Phase 1: UI 主题重新设计

#### 1.1 更新颜色系统
- 修改 `global.css` 中的 CSS 变量
- 主色从蓝紫色改为青色 (#0abfb6)
- 背景改为浅灰色 (#f5f7fa)
- 卡片改为纯白色

#### 1.2 重新设计页面头部
- 参考图片4：浅色头部或白色头部
- 标题左对齐，右侧操作按钮
- 移除渐变背景

#### 1.3 重新设计统计卡片
- 参考图片2：简洁的白色卡片
- 图表区域使用青色线条
- 数据展示更清晰

#### 1.4 重新设计账单列表
- 参考图片2/4：序号 + 描述 + 金额
- 简洁的列表项设计
- 移除复杂的多选样式

#### 1.5 重新设计按钮
- 参考图片3/4：青色圆角按钮
- 统一按钮样式

### Phase 2: OCR 识别算法修复

#### 2.1 修复日期识别逻辑
- 修改 `extractDate` 函数
- 优先从 OCR 文本中提取日期
- 支持多种日期格式（微信、支付宝、银行）
- 每条记录应有自己的日期，而不是统一使用当前日期

#### 2.2 修复数据条数统计
- 修改 `parseBillTextWithAutoMatch` 函数
- 确保每张图片的识别结果独立计算
- 修复 `getStartIndex` 函数的逻辑错误

#### 2.3 修复批量识别逻辑
- 修改 `Import/index.tsx` 中的处理逻辑
- 确保多张图片识别时数据不混乱
- 每张图片的识别结果应该正确分组显示

#### 2.4 优化识别结果展示
- 在 Import 页面清晰展示每张图片的识别结果
- 允许用户编辑每条记录的日期
- 显示识别到的原始日期信息

### Phase 3: 具体文件修改

#### 修改文件列表

1. **src/styles/global.css**
   - 更新 CSS 变量颜色值
   - 重新设计卡片、按钮、列表样式

   - 重新设计统计卡片布局
   - 简化账单列表样式
   - 更新头部设计

3. **src/pages/Import/index.tsx**
   - 优化识别结果展示
   - 添加日期编辑功能
   - 修复批量识别逻辑

4. **src/services/ocr.ts**
   - 修复日期提取逻辑
   - 优化金额识别
   - 确保每条记录有正确的日期

5. **src/services/billFormatRecognizer.ts**
   - 优化格式识别
   - 改进日期解析

6. **src/components/BottomNav.tsx**
   - 更新导航样式匹配新主题
